git命令的使用

git最大的功能是管理代码

①创建仓库名，在网页上创建。（clear）命令是清除当前命令行的显示。

cd ..(上一步）

cd projectName/（进入这个文件夹，事先在本地创建好的文件夹）

git init（版本库）

git status（查询状态）红色

git add .（添加到缓存）

git status/git commit -m 'init'（-m后面的是说明）

git status（查询状态）

git commit -m 'init'第一次提交到分支上

git status（查询状态）

git remote add origin https://github.com/username/projectName.git

git push -u origin(远程) master(主机)（提交到）

username:;userpassword:;

②链接：分配权限：settings-->collborators-->右侧：账号名称填写

分支操作

git clone 地址(粘贴)

有改动进入项目目录查询状态后执行下面语句

git add .-->git commit -m ""-->git push -u origin master

----git pull从网上弄下来

(master|merging)有冲突

工作时全放在master会有问题，需建立分支

git branch lee李分支
git branch查询分支
git checkout lee切换分支
git merge lee合并
git branch -d lee删除分支

如果用master分支，仓库必须起名叫`sensui7.github.io`，访问页面为http://sensui7.github.io

如果你想随便起名字，也可以，不过只能用`gh-pages`分支，比如你的blog仓库的`gh-pages`分支是博客内容，则访问页面为http://sensui7.github.io/blog/
