git命令的使用

git最大的功能是管理代码

命令含义

git clear清除当前命令行的显示
cd ..上一步
git clone 地址(粘贴) 克隆
git pull网上的更新到本地
subl .启动sublime快捷键win下不可以！
ls -a隐藏文件你给我出来！
Already up-to-date已更新的意思，不是命令。

首次在github.com上创建项目

①在网上创建仓库名
②本地创建文件夹与网上的仓库名一致
③cd projeceName
④git init版本库
⑤git add .添加到缓存
⑥git statu查询状态红色
⑦git status/git commit -m 'init' -m后面的是说明
⑧git status查询状态绿色
⑨git commit -m 'init'第一次提交到分支上
⑩git remote add origin https://github.com/username/projectName.git首次提交
git push -u origin(远程) master(主机)（提交到）
username:;userpassword:;

分配权限

settings-->collborators-->右侧：账号名称填写

工作时全放在master会有问题，需建立分支

git branch查看分支
git branch gh-pages新建分支
git checkout gh-pages切换分支
git merge gh-pages分支内容合并
git branch -d lee删除分支
git checkout还有一个撤销的功能 index.html
git commit -am "gh-pages"省去add命令
git pull master同步到本地
将gh-pages上面的内容同步到master上面----git checkout master----git merge gh-pages
将主分支的弄到分支上---git checkout gh-pages---git pull origin master

注意事项

1.如果用master分支，仓库必须起名叫sensui7.github.io 访问页面为http://sensui7.github.io
2.如果你想随便起名字，也可以，不过只能用`gh-pages`分支，比如你的blog仓库的`gh-pages`分支是博客内容，则访问页面为http://sensui7.github.io/blog/
3.md文件不见时需输入命令将网上的同步下来git pull --rebase origin master

第二次创建仓库

①网上创建仓库名
②本地执行git clone 网址----到达本地后是个空文件夹相当于在本地新建文件夹
③本地创建READMD.md
④git add -A
⑤git commit -a -m 'readom'
⑥git config --global user.email ""
git config -- global user.name ""
git checkout -b gh-pages
git push -u origin gh-pages
